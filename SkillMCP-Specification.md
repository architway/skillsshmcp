# 🧠 SkillMCP: Core Architecture & Implementation Guide (Online Mode)

## 📌 1. Project Overview
SkillMCP is a Just-In-Time (JIT) Model Context Protocol (MCP) server. It connects AI coding agents (Claude Code, Gemini CLI) directly to the `skills.sh` registry. It prevents context bloat by bypassing massive system prompts and injecting highly specific, community-vetted instructional markdown files (`SKILL.md`) only when the agent explicitly requests them.

This architecture is **Strictly Online** and **Stateless**, relying on LLM semantic reasoning and Chat Transcript Anchoring for state management.

---

## ⚙️ 2. Internal Engine Modules (The Backend Features)
These are the internal Python/TypeScript functions the MCP server runs. The AI does *not* see these directly; they act as the middleware pipeline.

### 🧩 Module A: `QueryExpansionEngine`
* **Purpose:** Bypasses Vercel's rigid string-matching search.
* **Logic:** Accepts an array of up to 4 kebab-case keywords generated by the AI (e.g., `["ml", "machine-learning"]`).
* **Output:** Prepares parallel search tasks for the API Client.

### 🌐 Module B: `VercelAPIClient` (The Walkie-Talkie)
* **Purpose:** Fetches raw data without headless browsers.
* **Logic:** Executes asynchronous `HTTP GET` requests to the `skills.sh` backend API (or executes `subprocess.run(['npx', 'skills', 'find', '--json'])` as a fallback).
* **Execution:** Runs the array from Module A concurrently to minimize latency. Returns raw JSON arrays of skill metadata.

### 🧮 Module C: `RankerAndDeduplicator` (The 30:70 Filter)
* **Purpose:** Cleans the messy API data and filters out viral bloat.
* **Logic:**
  1. **Merge & Deduplicate:** Flattens the concurrent search results and removes duplicate `skill_names`.
  2. **The 30:70 Algorithm:** Calculates a composite score for each skill: 
     `score = (installs * 0.3) + (github_stars * 0.7)`
  3. **Truncate:** Sorts the list by the composite score in descending order and slices the `Top 3` results.
* **Output:** A highly compressed, clean JSON list containing only the `skill_name` and `description` of the 3 best options.

## 🛠️ 3. Exposed MCP Tools (The AI Interface)
These are the exact JSON-RPC schemas exposed to the AI Host. 

### Tool 1: `search_skills_online`
* **Description:** Used when the AI needs an architectural pattern but doesn't know the exact skill name.
* **Prompt Engineering Mandate:** The AI is strictly instructed to generate 4 distinct, kebab-case search variations based on its intent.
* **Internal Trigger:** Passes the AI's array to `Module A -> Module B -> Module C`.
* **Return Value:** Returns the Top 3 filtered skills to the AI.

### Tool 2: `fetch_skill_content`
* **Description:** Used to download the actual `SKILL.md` payload. Triggered after the AI selects a winner from `search_skills_online`, or when reloading a known skill.
* **Internal Trigger:** Makes a direct `HTTP GET` to fetch the raw markdown file for the specified `skill_name`.
* **Return Value:** The complete markdown text, ready for context injection.

## 📝 4. UI/UX & State Management (The "Tattoo" System)
SkillMCP uses zero local databases (No SQLite, No JSON stores). It leverages the CLI's native chat transcript for persistence and collision avoidance.

### The System Instruction (Prompt Injection)
Upon connecting to the MCP, the AI receives this absolute rule:
> **CRITICAL MANDATE:** Whenever you successfully use `fetch_skill_content`, your VERY FIRST line of output to the user MUST be the following attribution badge: 
> `🛠️ [Active Skill: <skill-name>]`
> 
> **STATE MANAGEMENT RULES:**
> 1. **Continuity:** If you see this badge in the chat history, assume the skill is active. 
> 2. **Reloading:** If you lose context of the skill but see the badge, use `fetch_skill_content(<skill-name>)` to instantly reload it without searching.
> 3. **Reworks (The Skip Logic):** If the user states the code failed or requests a completely different architecture, DO NOT reload the tattooed skill. Initiate a fresh `search_skills_online` and deliberately select a different architectural path.

---

## 🛡️ 5. Fallbacks & Error Handling
* **Bad AI Input:** If the AI passes a single string instead of an array to `search_skills_online`, the server throws an MCP Error: *"Validation Failed: Must provide an array of search variations."*
* **API Timeout:** If Vercel's API is down, returns an MCP Error: *"Registry Offline. Please try again later."*
* **Zero Results:** If the search yields nothing, returns an MCP Error: *"No skills found. Please revise your search array with broader kebab-case terms."*
